# Feature Docs Index

Last updated: 2026-03-30

## Purpose

This folder contains feature-level documentation.

Use it when you need one place that explains:

1. what a feature is for
2. which workflows it owns
3. which code areas implement it
4. which tables and contracts it touches
5. which other features it depends on

## Numbering rule

Feature docs use gaps in numbering so related future docs can be inserted later without renaming everything.

Suggested ranges:

- `10-19` identity and onboarding
- `20-29` acquisition and scraping
- `30-39` notebook, opportunities, and application workflow
- `40-49` documents and extraction
- `50-59` career profile and matching
- `60-69` support and ops surfaces

## Required sections for every feature doc

1. Purpose
2. User value
3. Scope
4. Main workflow
5. Code areas
6. Data model
7. APIs/events
8. Dependencies
9. Related docs
